There is a wiring step that is easy to miss and fails *silently* when you do: the listener beans have to be registered, and `@TransactionalEventListener` needs one extra bean before Spring will honour it.

== grails-app/services/ does not auto-register a class unless it ends in `Service`

Grails turns a class under `grails-app/services/` into a Spring bean only when its name ends in `Service`. Our listeners are named `...Listener`, so Grails does *not* register them. A class that is never a bean is never scanned by Spring's `EventListenerMethodProcessor`, so its `@EventListener` / `@TransactionalEventListener` methods are never wired - and nothing warns you. The publisher calls `publishEvent(...)`, no listener runs, and the only symptom is a side-effect that quietly never happens.

== `@TransactionalEventListener` needs a `TransactionalEventListenerFactory` bean

Registering the bean is enough for a *plain* `@EventListener` to fire. It is **not** enough for `@TransactionalEventListener`. The transaction-phase-bound variant is only processed when a `TransactionalEventListenerFactory` bean exists in the context. In a standard Spring Boot application `@EnableTransactionManagement` contributes that factory automatically; in this Grails application it is not registered by default, so every `@TransactionalEventListener` is silently ignored until you declare it yourself - which is the last bean in the `resources.groovy` above.

The failure modes are worth committing to memory because they are all silent:

* *Listener class not a `Spring` bean* (wrong suffix, not in `resources.groovy`) -> neither `@EventListener` nor `@TransactionalEventListener` fires.
* *Bean registered but no `TransactionalEventListenerFactory`* -> plain `@EventListener` fires, every `@TransactionalEventListener` is ignored. This is the confusing one: the synchronous orchestration listener in the second half of this guide works, while the AFTER_COMMIT listeners appear dead.
* *Both in place* -> everything fires.

A fast way to confirm the wiring during development is to drop a `log.error('listener fired')` as the first line of a listener and watch for it when a transaction commits.
