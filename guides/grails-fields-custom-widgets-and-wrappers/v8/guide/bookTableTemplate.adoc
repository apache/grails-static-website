Override the whole table for `Book` so that:

* The table renders as a Bootstrap 5 striped, hover-enabled table.
* The header row uses `<th scope="col">` for accessibility.
* The last column always contains action buttons (View / Edit / Delete) without you adding a property to `properties=`.

The model variables passed to the table template are:

[cols="1,1,3"]
|===
| Variable | Type | Description

| `collection`
| `Collection<?>`
| The list of beans to render.

| `domainClass`
| `Class<?>`
| The Java class of the elements - useful when emitting links via `<g:link controller="...">`.

| `domainProperties`
| `List<DomainProperty>`
| The columns the plugin selected, already filtered by the `properties=`/`except=` attributes.
|===

Inside the loop, render each cell with `<f:displayWidget bean="${book}" property="${prop.name}"/>`. That dispatches to whichever `_displayWidget.gsp` is most specific - your `Date` display widget for the `publishedDate` cell, your `Boolean` widget for the `inStock` cell, and so on.

`<f:table>` for any _other_ domain class still uses the default table template. Only `Book` gets this customised layout.
